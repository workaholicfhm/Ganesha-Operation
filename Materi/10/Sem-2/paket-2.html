Algoritma dan Pemrograman 1

2.1.1 Algoritma

Algoritma adalah suatu kumpulan instruksi terstruktur dan terbatas yang dapat diimplementasikan dalam bentuk program komputer untuk menyelesaikan suatu permasalahan.

Merupakan strategi atau "resep" untuk menyelesaikan suatu masalah.

Ada dua cara:

  • Diagram alir
  • Pseudocode

A. Diagram Alir

Dibuat dalam bentuk aliran simbol yang dapat ditelusuri dari suatu titik hingga titik akhir dari program.

Simbol standar ANSI/ISO:

  • Garis alir → arah yang menunjukkan aliran program dari awal hingga akhir
  • Terminator → titik awal / akhir suatu program
  • Proses → kegiatan komputasi yang dilakukan oleh program
  • Keputusan → titik percabangan yang salah satu cabangnya dapat dilalui oleh program berdasarkan kondisi
  • Masukan/Keluaran →
  • Subprogram → kegiatan atau proses lain yang telah didefinisikan sebelumnya
  • Penghubung dalam halaman → menghubungkan suatu titik pada diagram alir ke titik lain pada halaman yang sama
  • Penghubung antar halaman → menghubungkan suatu titik pada diagram alir pada halaman lain yang berbeda

Diagram alir menghitung luas persegi

flowchart TB n1(["start"]) --> n3["READ sisi"] n4["PRINT luas"] --> n2(["end"]) n3 --> n5["luas = sisi * sisi"] n5 --> n4 

input : Nilai panjang sisi persegi
output : Luas persegi

input sisi
luas <= sisi * sisi
print luas

2. Diagram alir menentukan hasil pembagian

flowchart TB n1(["Start"]) --> n2["READ pembilang, penyebut"] n2 --> n3["penyebut = 0?"] n3 -- No --> n4["Hasil = pembilang/penyebut"] n3 -- Yes --> n6["PRINT penyebut tidak boleh nol"] n4 --> n5["PRINT Hasil"] n6 --> n7(["END"]) n5 --> n7 

Input : pembilang, penyebut
Output : hasil pembagian tercetak

input pembilang, penyebut
if penyebut = 0
print "penyebut tidak boleh kosong"

else
hasil <= pembilang/penyebut
print hasil

3. Diagram alir menghitung mundur dari N hingga 1

flowchart TB n1(["Start"]) --> n2["READ N"] n2 --> n3["N > 0"] n3 -- Yes --> n4["PRINT N"] n3 -- No --> n6(["END"]) n4 --> n5["N = N-1"] n5 --> n3

B. Pseudocode

Pseudocode (kode semu atau kode pseudo) adalah suatu bahasa buatan manusia yang sifatnya informal untuk mempresentasikan algoritma.

Untuk menutupi kekurangan diagram alir dalam mempresentasikan konsep-konsep pemrograman.

Hal yang perlu diperhatikan:

  • Satu baris hanya untuk satu statement
  • Indentasi dalam menuliskan pernyataan

B. Struktur Program Bahasa C

Baris pertama digunakan untuk memasukkan header file stdio.h.

Baris 2–5 implementasi fungsi main, yang akan dijalankan pertama kali dalam program.

a. Kata kunci

Keyword adalah kata yang sudah memiliki makna khusus yang tidak dapat diubah oleh pemrogram.

Terdapat 32 kata kunci standar pada bahasa C.

b. Identifier

Nama unik yang dapat diingat dan diberikan pada program C → variabel dan fungsi.

Aturan:

  • Tidak boleh sama dengan kata kunci
  • Disusun dari kombinasi huruf besar dan kecil, angka, atau underscore
  • Harus dimulai dengan huruf atau underscore
  • Case-sensitive, beda penulisan, beda identifier

c. Tipe data

Nilai yang dimasukkan ke dalam komputer akan disimpan dalam bentuk biner, oleh karena itu perlu memberi tahu program tentang data kita.

Tipe data di bahasa C.

d. Variabel

Nilai yang dimasukkan akan disimpan dalam memori, komputer memberikan alamat pada lokasi tersebut namun susah diingat oleh manusia, oleh karena itu kita butuh variabel.

e. Konstanta

Konstanta dapat dideklarasikan seperti variabel, dengan menambahkan kata kunci const di depan tipe data.

Nilai awal harus langsung diberikan.

Konstanta tidak dapat berubah nilai.

Setiap tipe data memiliki spesifikasi format yang dapat digunakan untuk menjelaskan jenis data kepada program.

%cmenampilkan karakter
%smenampilkan teks string
%d dan %imenampilkan bilangan desimal
%fmenampilkan bilangan pecahan
%omenampilkan bilangan oktal
%xmenampilkan bilangan heksadesimal
